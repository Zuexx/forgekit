================================================================================
KEY TECHNICAL DECISIONS
================================================================================

1. IMiddleware vs Traditional Middleware
   - Used IMiddleware interface for dependency injection
   - Registered as transient service in DI container

2. LogContext for Correlation ID
   - Uses AsyncLocal<T> for async-safe propagation
   - Automatically available to all logs within request context

3. Log Levels
   - Debug: Handler entry/exit, validation checks
   - Information: Business operations, state changes
   - Warning: Business rule violations
   - Error: Unhandled exceptions

4. Structured Properties
   - Named properties instead of string concatenation
   - Enables JSON parsing for log aggregation

================================================================================
LOGGING PATTERNS
================================================================================

Commands: Log at Information on success (business-significant)
Queries: Log at Debug on success (informational)
Services: Debug=validation, Info=operations, Warn=conflicts
Middleware: Correlation ID injected into all logs
